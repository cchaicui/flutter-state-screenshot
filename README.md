# Flutter State Screenshot

> 一句话说完：你跟 Cursor 说"帮我截图 XX 功能所有状态"，它就会自动分析代码、操控模拟器、逐状态截图（含极端/边界情况），还能在多个机型上对比截图，最后输出一个漂亮的 HTML 文档。

## 它能干什么？

开发 Flutter App 的时候，你肯定遇到过这些场景：

- 设计评审前，需要把一个功能的"所有状态"都截图出来给设计师看
- QA 要你提供"加载中、成功、失败、空态、非会员"等各种状态的截图
- 自己改完代码，想快速检查每个状态的 UI 是否正常
- **需要验证极端情况**：超长文本会不会溢出？空数据会不会崩？数字太大会不会撑爆布局？
- **多机型适配验收**：同一个页面在 Pro Max 大屏和 SE 小屏上表现一样吗？

**这个 Skill 就是帮你自动完成这件事。**

它会：
1. 读你的 Dart 代码，自动分析出所有状态变量和条件分支
2. 列出完整的状态清单（比如：已登录/未登录、VIP/非VIP、加载中/成功/失败...）
3. **主动构造极端数据**（空内容、超长文本、超大数字、图片404等），截取边界场景
4. 自动操控 iOS 模拟器，逐个状态截图
5. **支持多机型对比**：同一状态自动在 Pro / 标准 / SE 等不同尺寸设备上截图
6. 生成一个 HTML 文档，正常状态和极端场景分区展示，多机型横向对比，每张截图套手机壳

## 安装教程（3 步搞定）

### 前提条件

- macOS 系统（需要 Xcode 和 iOS 模拟器）
- 已安装 Flutter SDK
- 已安装 [Cursor IDE](https://cursor.sh)

### 第 1 步：下载 Skill 文件

```bash
git clone https://github.com/cchaicui/flutter-state-screenshot.git ~/skill/flutter-state-screenshot
```

确保文件夹结构是这样的：

```
flutter-state-screenshot/
├── SKILL.md      ← 核心文件，AI 读这个来工作
└── README.md     ← 你正在看的这个文档
```

### 第 2 步：在 Cursor 中注册 Skill

1. 打开 Cursor IDE
2. 按 `Cmd + Shift + P`，输入 `Skills`，选择 **Cursor Settings: Skills**
3. 点击 **Add Skill**
4. 填入 SKILL.md 的完整路径，比如：
   ```
   /Users/你的用户名/skill/flutter-state-screenshot/SKILL.md
   ```
5. 点确认

> 如果你找不到 Skills 设置，也可以手动编辑 Cursor 的 settings.json，添加：
> ```json
> "cursor.skills": [
>   "/Users/你的用户名/skill/flutter-state-screenshot/SKILL.md"
> ]
> ```

### 第 3 步：验证安装

在 Cursor 的 Agent 模式中，随便对它说：

```
帮我截图首页的所有状态
```

如果它开始分析你的代码并提到"状态维度"、"极端场景"、"模拟器截图"，说明安装成功了。

## 使用指南

### 基本用法

直接用自然语言告诉 Cursor 你想截图哪个功能：

```
帮我把登录页面的所有状态截图出来
```

```
截图一下设置页面的全部状态
```

```
帮我看看首页 Clone 视频功能的状态矩阵
```

### 触发关键词

以下说法都能触发这个 Skill：

| 你说的话 | 效果 |
|---------|------|
| "帮我截图 XX 功能所有状态" | 完整截图（含极端场景） |
| "帮我把 XX 全部状态列出来" | 先列清单再截图 |
| "截图 XX 页面" | 完整截图（含极端场景） |
| "XX 的状态矩阵输出给我" | 完整截图（含极端场景） |
| "帮我截图 XX 的极端情况" | 专注极端/边界场景 |
| "截图 XX 的边界 case" | 专注极端/边界场景 |
| "帮我在不同机型上截图 XX" | 多机型对比截图 |
| "大屏小屏都截一下" | 多机型对比截图 |
| "SE 上看看 XX 的效果" | 指定机型截图 |
| "帮我做一下 XX 的适配验收" | 多机型 + 全状态 |

### 进阶用法

**只截某些状态：**
```
帮我截图支付功能的加载中和失败状态
```

**指定优先级：**
```
截图首页所有状态，优先截这次改动涉及的
```

**只截极端场景：**
```
帮我截图用户列表的边界case：空列表、超长列表、超长用户名
```

**多机型对比：**
```
帮我在不同机型上截图首页所有状态
```

默认会在 iPhone 16 Pro / iPhone 16 / iPhone SE 三个机型上截图。你也可以指定：
```
帮我在 Pro Max 和 SE 上对比截图设置页
```

**状态太多时：**

如果一个功能有超过 15 个状态，AI 会先列出完整清单让你确认，你可以说：

```
只截 1、3、5、8 这几个状态
```

### 输出文件在哪里？

截图和 HTML 输出在 Skill 文件夹下的 `output/` 目录：

```
flutter-state-screenshot/
├── SKILL.md
├── README.md
└── output/                    ← 自动生成
    └── clone_video/           ← 按功能模块分文件夹
        ├── clone_states.html  ← 打开这个看结果
        └── screenshots/
            ├── 01_xxx.png          ← 正常状态
            ├── ...
            ├── 09_edge_xxx.png     ← 极端场景（edge_ 前缀）
            └── ...
```

多机型模式下，截图文件名会带机型标识：
```
screenshots/
├── 01_pro_plan_empty.png     ← iPhone 16 Pro
├── 01_std_plan_empty.png     ← iPhone 16
├── 01_se_plan_empty.png      ← iPhone SE
└── ...
```

HTML 文件可以直接用浏览器打开，也可以发给设计师/QA 查看。

## 工作原理（了解即可）

```
你说"截图XX功能"
       ↓
Phase 1: AI 读你的 Dart 代码 → 分析所有状态变量
       ↓
Phase 2: 检查/启动 iOS 模拟器 + Flutter app
         （多机型模式：准备多个模拟器）
       ↓
Phase 3: 逐状态截图
         ├── 优先通过 UI 操作（点击/滑动）切换状态
         ├── 必要时临时修改代码初始值
         ├── 主动构造极端数据（空/超长/零值/404等）
         ├── 每个状态用 xcrun simctl 截图
         └── 多机型模式：同一状态在每个机型上各截一张
       ↓
Phase 4: 生成 HTML 文档（正常 + 极端分区，多机型横向对比）
       ↓
Phase 5: 在浏览器中打开给你看
```

### 它会自动覆盖哪些状态？

**正常状态维度：**

| 维度 | 示例 |
|------|------|
| 认证/订阅 | 已登录/未登录、VIP/非VIP |
| 网络 | 成功/失败/超时/离线 |
| 加载 | loading/skeleton/shimmer |
| 数据 | 有数据/空态/首次加载 |
| 弹窗 | Dialog/BottomSheet/Toast |
| 权限 | 通知权限、相册权限 |
| 操作反馈 | Copy 成功、刷新、展开/折叠 |
| 外部唤起 | 深链接/剪贴板跳入 |

**极端 / 边界场景（自动覆盖）：**

| 维度 | 示例 |
|------|------|
| 内容长度 | 空字符串、极短(1字)、超长(200+字符)、多行(20+行) |
| 列表边界 | 0条、1条、恰好满屏、100+条、item高度不一致 |
| 数值边界 | 0、负数、极大数(999999999)、百分比≥100% |
| 图片媒体 | 无图(null)、加载失败(404)、异常宽高比 |
| 特殊字符 | emoji、中日韩混排、HTML标签 |
| 时间边界 | 刚刚、跨天、很久以前、未来时间 |
| 快速重复操作 | 连点按钮、防重复提交 |

**多机型对比（按需启用）：**

| 机型组 | 包含机型 |
|--------|---------|
| 核心三件套（默认） | iPhone 16 Pro、iPhone 16、iPhone SE |
| 完整覆盖 | + iPhone 16 Pro Max、iPhone 13 mini |
| 自定义 | 你指定哪些就截哪些 |

HTML 中同一状态横向排列所有机型截图，方便一眼看出适配差异。

## 常见问题

### Q: 截图时弹出 iOS 粘贴权限弹窗怎么办？

iOS 16+ 的 `Clipboard.getData()` 会触发系统弹窗。解决方法：

1. 截图前运行 `pbcopy < /dev/null` 清空剪贴板
2. 如果你的 app 有自动剪贴板检测逻辑，在 debug 模式下加个开关跳过它

### Q: 截图后我的代码被改了怎么办？

不用担心。AI 在修改代码截图前会用 `git stash` 保存你的改动，截完后用 `git checkout` 还原，最后 `git stash pop` 恢复。你的代码不会丢失。

### Q: 极端数据会不会影响我的真实数据？

不会。所有极端数据都是通过临时修改代码中的变量初始值来构造的，截图完成后代码会自动还原。不涉及任何数据库或后端数据。

### Q: 支持 Android 模拟器吗？

目前只支持 **iOS 模拟器**（通过 `xcrun simctl`）。Android 模拟器需要用 `adb` 系列命令，原理相同但尚未适配。

### Q: 截不到某些状态怎么办？

有些状态确实很难自动触发（比如需要特定后端数据、需要等待定时器触发等）。AI 会在 HTML 报告中标注这些状态为"需后端配合"或"代码预留"，方便你手动补充。

### Q: 可以用在非 Flutter 项目上吗？

理论上截图和 HTML 生成的逻辑是通用的，但状态分析部分是针对 Flutter/Dart 代码设计的。如果你是 Swift/SwiftUI 或 React Native 项目，状态分析部分需要自行适配。

### Q: 多机型截图要等很久吗？

每个机型需要启动模拟器 + 安装 app，大约额外 1-2 分钟。核心三件套（3 个机型）总共约 5-10 分钟，取决于状态数量。AI 会自动优化流程：先在主机型上截全部状态，再在其他机型上只截关键状态。

### Q: 我没装那么多模拟器怎么办？

AI 会自动用 `xcrun simctl create` 创建缺失的模拟器（只要你的 Xcode 有对应的 runtime）。不需要你手动创建。

### Q: 极端场景太多了，不想全截怎么办？

你可以明确告诉 AI：

```
只截正常状态，跳过边界case
```

或者指定只要某几类极端场景：

```
只截超长文本和空列表的边界case
```
